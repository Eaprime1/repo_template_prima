# Guides

Guides are practical documentation written as if they belong to the concept's world.
They teach real skills through the lens of the concept's lore and setting.

## Structure

```
guides/
  README.md           ← this file
  getting-started.md  ← the first guide new players read
  <topic>.md          ← add guides here as the concept grows
```

## Guide Philosophy

A guide is not a manual page. It explains **why** alongside **how**, and the
why is grounded in the concept's world.

- Write in the voice of the world — a mentor, an elder, a systems artifact.
- The practical content must be accurate and complete.
- Keep guides short enough to read in one sitting.
